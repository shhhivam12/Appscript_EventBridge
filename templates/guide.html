    User Guide

    Everything you need to connect ServiceNow and Telegram to Google Apps Script

    Getting Started in 5 Steps
    1
    Connect your Google Account

    Go to Settings and click Connect Google Account. This grants access to your Apps Script projects via the Google Apps Script REST API.

    The Apps Script API must be enabled in your GCP project for script execution to work.
    2
    Add a Credential

    Go to Credentials and add one for your external platform — ServiceNow, Telegram, or a custom app. Each credential generates a unique webhook URL.

    3
    Create a Workspace and Project

    Workspaces are top-level containers (e.g. "Production"). Projects group related workflows (e.g. "ServiceNow Integrations"). Navigate to Workspaces and create both.

    4
    Build a Workflow

    Open your project and click New Workflow. Set a Trigger (which credential + event type) and an Action (which Apps Script project + function to call). Functions are auto-loaded from your Google Drive.

    5
    Activate and Test

    Toggle the workflow Active, then use the Test Payload panel in the editor to fire a sample event. Results appear instantly — green means success, red shows the error.

    Test payloads are pre-filled with realistic data for your trigger type.
    How It Works
    External Event

    Setting Up ServiceNow Integration
    1
    Create OAuth Application in ServiceNow

    In ServiceNow, navigate to System OAuth → Application Registry. Click New → Create an OAuth API endpoint for external clients. Copy the Client ID and Client Secret.

    2
    Add ServiceNow Credential

    In Apps Script Bridge, go to Credentials → New ServiceNow. Fill in your instance URL, Client ID, and Client Secret. After saving, copy the Webhook URL from the manage page.

    3
    Create a Business Rule in ServiceNow

    Navigate to System Definition → Business Rules. Create a new rule on your target table (e.g., incident). Under Advanced tab, add this script:

    // Business Rule: Send to Apps Script Bridge
(function executeRule(current, previous) {
    var req = new sn_ws.RESTMessageV2();
    req.setEndpoint('YOUR_WEBHOOK_URL_HERE');
    req.setHttpMethod('POST');
    req.setRequestHeader('Content-Type', 'application/json');

    var body = {
        event_type:        'incident.created',
        number:            current.number.toString(),
        short_description: current.short_description.toString(),
        priority:          current.priority.toString(),
        state:             current.state.toString(),
        caller_id:         current.caller_id.getDisplayValue(),
        sys_id:            current.sys_id.toString()
    };

    req.setRequestBody(JSON.stringify(body));
    var response = req.execute();
})(current, previous);
    Replace YOUR_WEBHOOK_URL_HERE with the URL from your credential manage page.
    4
    Create a Workflow in Apps Script Bridge

    Create a workflow using your ServiceNow credential. Set the event type to match what you send in the Business Rule (e.g., incident.created). Select your Apps Script function.

    Supported Event Types
    incident.createdNew incident created
    incident.updatedIncident fields changed
    incident.resolvedIncident moved to resolved
    change_request.createdNew change request
    change_request.approvedChange request approved
    problem.createdNew problem record
    customAny other event type you define
    Security Notes
    !
    Webhook URL contains the credential ID

    Keep your webhook URL private — it's the only authentication for ServiceNow events.

    !
    Deactivate to stop all events

    If compromised, deactivate the credential in Apps Script Bridge — the webhook will immediately return 404.

    Troubleshooting
    ?
    Events not arriving

    Check the Business Rule is active, the endpoint URL is correct, and the ServiceNow instance can reach your server. Use ngrok for local dev.

    ?
    Workflow shows "skipped"

    The event_type in the payload doesn't match the workflow's configured event type. Check Event Logs to see the actual value received.

    Setting Up Telegram Bot Integration
    1
    Create a Telegram Bot

    Open Telegram and search for @BotFather. Send /newbot, follow the prompts to name your bot, and copy the Bot Token (format: 123456789:ABC-DEF1234…).

    2
    Add Telegram Credential

    Go to Credentials → New Telegram. Paste your Bot Token and save.

    3
    Expose Your Server Publicly

    Telegram requires a public HTTPS URL to deliver updates. For local development, use ngrok:

    ngrok http 5000

    Copy the https://xxxx.ngrok.io URL and update your .env file:

    BASE_URL=https://xxxx.ngrok.io

    Then restart your Flask server.

    For production, deploy behind a proper HTTPS domain and set BASE_URL accordingly.
    4
    Register the Webhook

    Open your Telegram credential's manage page and click Register Webhook with Telegram. This calls the Telegram API to point your bot at Apps Script Bridge's webhook URL.

    Click Check Status to verify registration succeeded.

    5
    Create a Workflow

    Create a workflow using your Telegram credential. Choose the event type: message for regular messages, command for bot commands (e.g. /start), or callback_query for inline button presses.

    Supported Event Types
    messageAny regular text message
    commandBot command (starts with /)
    callback_queryInline keyboard button press
    edited_messageUser edited a message
    inline_queryInline mode query
    Troubleshooting
    ?
    Webhook registration fails

    Your BASE_URL must be an HTTPS URL. Run ngrok and update your .env, then restart before registering.

    ?
    Messages not arriving

    Check the webhook is registered (Check Status shows the URL). If BASE_URL changed (e.g. new ngrok tunnel), re-register the webhook.

    ?
    command events not firing

    Make sure your workflow event type is set to command, not message — Apps Script Bridge automatically classifies messages starting with / as commands.

    Writing Functions for Apps Script Bridge

    Apps Script Bridge calls your function using the Apps Script REST API, passing the full event payload as the first parameter.

    Basic Function Template
    /**
 * Called by Apps Script Bridge when a ServiceNow incident is created.
 * @param {Object} payload  The full event payload from ServiceNow.
 */
function handleIncidentCreated(payload) {
  Logger.log('Received event: ' + payload.event_type);
  Logger.log('Incident number: ' + payload.number);

  // Example: write to Google Sheets
  var sheet = SpreadsheetApp.openById('SHEET_ID')
                             .getActiveSheet();
  sheet.appendRow([
    new Date(),
    payload.number,
    payload.short_description,
    payload.priority
  ]);
}
    Telegram Message Handler
    /**
 * Called when a Telegram message is received.
 * @param {Object} update  The full Telegram Update object.
 */
function handleTelegramMessage(update) {
  var msg  = update.message;
  var text = msg.text;
  var from = msg.from.first_name;

  Logger.log(from + ' said: ' + text);

  // Example: send a reply using UrlFetchApp
  // (Telegram bot token needed here)
  var token = PropertiesService.getScriptProperties()
                                .getProperty('TG_TOKEN');
  UrlFetchApp.fetch(
    'https://api.telegram.org/bot' + token + '/sendMessage',
    {
      method: 'post',
      contentType: 'application/json',
      payload: JSON.stringify({
        chat_id: msg.chat.id,
        text: 'Hello, ' + from + '! You said: ' + text
      })
    }
  );
}
    Requirements
    !
    Apps Script API enabled

    In Google Cloud Console → Library, enable "Apps Script API" for your GCP project.

    !
    Script linked to GCP project

    In the Apps Script editor, go to Project Settings → Google Cloud Platform project → link to the same GCP project as your OAuth credentials.

    !
    Function must be top-level

    The function must be a top-level function name() declaration in a .gs file — not nested inside another function or class.

    Auto-loading
    1
    Connect Google first

    Settings → Connect Google Account to authorise the Drive + Apps Script APIs.

    2
    Scripts auto-load

    All your Apps Script files from Google Drive appear in the Project dropdown automatically — no Script ID to copy.

    3
    Functions auto-load

    Select a project and all function foo() definitions are parsed and listed — no need to type function names manually.

    Common Issues
    !
    Apps Script returns 403

    The Apps Script API is not enabled, or the script isn't linked to the correct GCP project. Check both in Google Cloud Console.

    !
    Telegram webhook returns "Bad Request"

    BASE_URL must be a valid HTTPS URL. http://localhost:5000 is not accepted by Telegram — use ngrok.

    !
    Event log shows "skipped"

    Either no active workflow matches this credential + event_type, or the workflow is toggled inactive.

    !
    Function dropdown is empty

    Make sure your Google account is connected (Settings). If connected, the function must be a top-level function declaration in a .gs file.
